# Versioning And Rollout

Use this reference before committing, previewing, or publishing.

## Rules

- Work on branches
- Keep commits focused
- Preserve a clean baseline on `main`
- Prefer local-first changes
- Validate visually before live publishing

## Publishing path

1. Local change
2. Review diff
3. Commit
4. Preview on draft theme
5. Approve
6. Publish only when explicitly authorized

## Rollback

- Revert the last focused commit
- Or drop the feature branch if the iteration is rejected
