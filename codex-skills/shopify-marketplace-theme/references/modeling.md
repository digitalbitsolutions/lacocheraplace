# Services And Providers Modeling

Use this reference when redesigning product data or service detail UX.

## Service attributes to consider

- Duration
- Location
- Provider
- Coverage area
- Booking mode
- Preparation notes
- Pricing mode

## Provider attributes to consider

- Workshop name
- Specialty
- Area served
- Reputation or trust markers
- Service list
- Contact or booking path

## Principle

Do not force a perfect backend model too early. Start with a clear frontend representation that can later map to metafields or richer entities.
